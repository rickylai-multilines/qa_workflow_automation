Manage WIP for Each Team

Work-in-progress (WIP) is configured per team (department). Each team has WIP types by lead time (min–max days). Each WIP type has checkpoints with a critical date – the date by which the process must be actioned. The daily alert email lists checkpoints reaching the critical date and reminds users to act to avoid missing CRD.

How to manage WIP

  • WIP types define categories by lead time (min–max days) per team. They must be active to appear on the WIP dashboard.
  • Checkpoints belong to a WIP type and define stages (e.g. "Sample approval", "PP meeting") with rule type (CRD offset or previous checkpoint offset) and offset days. The computed date is the critical date – action by this date to avoid delaying shipment.
  • Each user sees only WIP for orders they created/modified (assigned via SOMain User ID). Supervisors see department-wide WIP.
  • Use the links below to open Django Admin to add or edit WIP types and checkpoints for each team.
  • After changing definitions, run sync_wip_orders to update WIP orders and tasks.
